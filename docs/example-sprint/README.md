# Example Sprint

This is a **sanitized example** of a real sprint execution using the multi-agent pipeline. It demonstrates how items flow through the 8 rounds, how review findings are structured, and what a sprint retrospective looks like.

The scenario: a SaaS product improving its dashboard and adding search/filter functionality to a list page.

## Files

- **[SPRINT.md](SPRINT.md)** — Sprint board snapshot showing items at different pipeline stages
- **[r5-findings.md](r5-findings.md)** — R5 review findings with severity levels and fix tracking
- **[retrospective.md](retrospective.md)** — Gate 7 retrospective with velocity and improvement items

## Key Things to Notice

1. **Pipeline tiers in action** — the sprint has both Fast Track (S) and Full Pipeline (M/L) items
2. **Parallel execution** — Wave 2 items run in parallel because they don't share files
3. **R5 findings format** — deduplicated, severity-labeled, with file:line references and fix status
4. **Definition of Done tracking** — each item has the full DoD checklist
5. **Context handoff** — handoff notes are 2-5 bullet summaries, not full agent output
